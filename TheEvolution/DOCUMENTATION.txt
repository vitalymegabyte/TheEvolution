
     ДОКУМЕНТЭЙШН
     WorkPlace - рабочий класс. В нем происходит всё взаимодействие с миром. Методы Start и Update работают аналогично движку Unity,
     то есть Start вызывается при запуске, Update вызывается каждый кадр. На данный момент задержки никакой нет, возможно,
     добавится позже.

     Класс Environment
     Отвечает за графическую (и не только) среду.

     Методы:

     drawPixel(x, y, r, g, b) - нарисовать пиксель в координатах x и y, цвета по rgb палитре r, g, и b соответственно.
     
     cycleX(x) - передаем x-координату, он возвращает ее нормализованной. т.е. например при рождении на клетке i-1 при i=0
     можно попасть в -1 элемент массива, а при рождении на клетке i+1 можно родить на клетке больше размера мира, что опять-
     таки нехорошо. С помощью этих методов можно "зациклить" мир.

     cycleY(y) - аналогично предыдущему.

     Environment(width, height) - конструктор среды. Передается ее ширина и высота.

     Переменные:

     WORLD_WIDTH - ширина мира, readonly
     WORLD_HEIGHT - высота мира, readonly

     Растягивается автоматически. Пикселизация должна быть красивой, это уже настроено :)

     Чтобы работать со средой необходимо передать ее в класс App. В методе Start это делается так:
     ((App)(App.Current)).Environment = environment;
     (Я правда не знаю, почему среду нельзя сразу сделать одну. Это не костыль, а возможность для будущих расширений, ибо 
     пытливые умы, к которым попадет исходник могут что-нибудь интересное придумать и с этим :) )

     ДЕМО
     Демо представляет собой просто "тьму, застилающую экран". Клетка каждый кадр при наличии свободного места "рожает" другую.
